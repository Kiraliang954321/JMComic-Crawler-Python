# OWASP Top 10 安全检查清单（2021 版）

## A01. 访问控制失效（Broken Access Control）
- [ ] API 路由是否都有权限校验？（不允许裸接口）
- [ ] 是否存在越权风险？用户能否访问其他用户的数据（IDOR）？
- [ ] 是否对请求中的资源 ID 做了归属校验？
- [ ] CORS 配置是否过于宽松（`Access-Control-Allow-Origin: *` 且携带凭证）？
- [ ] 是否存在路径遍历风险（如 `../../etc/passwd`）？
- [ ] JWT/会话令牌的权限声明是否在服务端校验？

## A02. 加密失效（Cryptographic Failures）
- [ ] 密码是否使用 bcrypt / argon2 等强哈希算法？（禁止 MD5 / SHA1）
- [ ] 敏感数据是否通过 HTTPS 传输？是否强制 HSTS？
- [ ] 加密密钥是否安全存储？（禁止硬编码，使用环境变量或密钥管理服务）
- [ ] 是否使用了已知不安全的加密算法（DES、3DES、RC4、ECB 模式）？
- [ ] 随机数生成是否使用了 `crypto.randomBytes` 而非 `Math.random()`？
- [ ] 证书验证是否被禁用（如 `NODE_TLS_REJECT_UNAUTHORIZED=0`）？

## A03. 注入攻击（Injection）
- [ ] SQL 查询是否使用参数化查询或 ORM（Prisma）？是否存在字符串拼接？
- [ ] 是否存在命令注入风险？（`exec` / `spawn` 拼接用户输入）
- [ ] NoSQL 查询是否做了参数过滤（如 MongoDB `$where` / `$regex` 注入）？
- [ ] LDAP / XPath / 模板引擎是否对用户输入做了转义？
- [ ] 用户输入是否在服务端做了验证和过滤？
- [ ] ORM 的原生查询（raw query）是否同样做了参数化处理？

## A04. 不安全设计（Insecure Design）
- [ ] 关键业务流程是否有速率限制（登录、注册、短信发送）？
- [ ] 敏感操作是否有二次确认机制（删除、修改密码、转账）？
- [ ] 分页查询是否限制了 `pageSize` 最大值？
- [ ] 用户输入是否有服务端校验？（不能只依赖前端验证）
- [ ] 是否存在过度数据暴露？（API 返回了前端不需要的敏感字段）
- [ ] 批量操作接口是否有数量上限？

## A05. 安全配置错误（Security Misconfiguration）
- [ ] 生产环境是否关闭了调试模式/详细错误堆栈？
- [ ] 默认账户/密码是否已修改？
- [ ] 不必要的 HTTP 方法（PUT、DELETE、TRACE）是否已禁用？
- [ ] 目录列表功能是否已关闭？
- [ ] 是否设置了安全的响应头（CSP、X-Content-Type-Options、X-Frame-Options）？
- [ ] 第三方服务的默认配置是否经过了安全审查（如 MinIO、Redis 无密码访问）？

## A06. 易受攻击和过时的组件（Vulnerable and Outdated Components）
- [ ] `npm dependencies` 中是否有已知漏洞？（定期 `npm audit`）
- [ ] 依赖库是否及时更新？是否存在长期不维护的库？
- [ ] 是否引入了不必要的第三方依赖？
- [ ] Node.js / 运行时版本是否存在已知安全漏洞？
- [ ] 前端 CDN 引入的第三方脚本是否固定版本且来源可信？

## A07. 身份认证失效（Identification and Authentication Failures）
- [ ] 密码是否明文存储？（必须使用 bcrypt / argon2 等加密）
- [ ] 登录失败是否有次数限制和账户锁定机制？
- [ ] 会话令牌是否使用安全的随机数生成？是否有过期时间？
- [ ] JWT 是否校验了签名算法（禁止 `none` 算法）和过期时间（`exp`）？
- [ ] 密码重置流程是否安全？（令牌一次性使用、有时效性）
- [ ] 是否支持多因素认证（MFA）用于敏感操作？

## A08. 软件和数据完整性故障（Software and Data Integrity Failures）
- [ ] 反序列化是否使用了安全的库？是否对反序列化数据做了类型校验？
- [ ] CI/CD 管道是否有代码签名或完整性校验？
- [ ] 第三方库更新是否经过完整性验证（lockfile + integrity hash）？
- [ ] 文件上传是否校验了文件类型、大小，并存储到隔离位置？
- [ ] 客户端数据（如 JWT payload、localStorage）是否做了防篡改校验？

## A09. 安全日志记录和监控失效（Security Logging and Monitoring Failures）
- [ ] 登录成功/失败、权限变更、敏感操作是否有日志记录？
- [ ] 日志中是否包含明文密码、令牌等敏感信息？
- [ ] 日志是否有防篡改和保留策略？
- [ ] 是否有异常行为的告警机制（暴力破解、异常 IP、大量 403）？
- [ ] 用户操作是否有审计追踪（谁在什么时间做了什么）？

## A10. 服务端请求伪造（SSRF）
- [ ] 用户提供的 URL 是否在服务端直接请求？是否有白名单限制？
- [ ] 是否禁止访问内网地址（`127.0.0.1`、`10.0.0.0/8`、`169.254.169.254` 等）？
- [ ] HTTP 客户端是否禁用了重定向跟随？或对重定向目标做了二次校验？
- [ ] 文件导入/URL 获取功能是否限制了协议（只允许 HTTPS，禁止 `file://`）？
- [ ] Webhook / 回调功能的目标地址是否经过验证？
